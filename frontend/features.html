Currently Implemented Features

The system extracts 15 robust URL-based lexical and structural features for real-time phishing detection.

IP Address in URL

Using an IP instead of a domain name is a strong phishing indicator.

URL Length

Abnormally long URLs are often used to hide suspicious parts.

Domain Length

Unusual domain length can indicate auto-generated or malicious domains.

Path Length

Deep and complex URL paths can be a sign of obfuscation.

URL Depth

Counts the number of sub-directories in the URL path.

Subdomains

Multiple sub-domain levels are used to mimic legitimate URLs.

HTTPS Usage

Checks for secure protocol; absence suggests an insecure connection.

Special Characters

High count of special chars signals potential obfuscation attempts.

Suspicious Keywords

Detects words like 'login', 'update', or 'secure' often used in phishing.

URL Entropy

Measures the randomness of characters in the entire URL.

Domain Entropy

Calculates randomness specifically within the domain portion.

Redirect Count

Multiple redirections obscure the final malicious destination.

Hyphen Analysis

Domains with multiple hyphens often mimic legitimate sites.

@ Symbol Detection

Forces browsers to ignore everything before it, masking the real domain.

Numeric Character Analysis

Anomalous numbers of digits can indicate machine-generated URLs.
